# Tiny Collect Prototype GDD Fixture

A bounded local 2D fixture where the player starts on a small tilemap, collects one visible goal marker, and exits through a declared trigger. The fixture uses placeholder/local manifest references only and exists to demonstrate evidence-gated GDD-to-prototype validation, not autonomous unrestricted game creation.

Acceptance intent:
- player start is visible and bounded;
- goal marker can be reached by local movement rules;
- exit trigger records a successful scenario verdict;
- unsupported or missing requirements remain explicit evidence gaps.
